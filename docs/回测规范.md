# 回测规范

本文档定义A股量化策略的回测工作流程和标准。

## 1. 回测类型

### 1.1 批量回测 (Batch Backtest)

**用途**: 在多个股票上测试策略，验证策略的普适性

**使用场景**:
- 策略开发验证
- 最终测试

**命令**:
```bash
python backtest/strategy_backtest/batch_backtest_V3.py -l 100 --strategy 策略模板 --start 20240101 --end 20241231
```

**说明**: 使用 `--limit` 从数据库随机获取100只股票进行回测

---

### 1.2 参数扫描 (Parameter Sweep)

**用途**: 系统性搜索最优参数组合

**使用场景**:
- 参数优化阶段
- 寻找策略的最佳阈值

**命令**:
```bash
python backtest/strategy_backtest/batch_backtest_V3.py -l 100 --param-grid "threshold:6,8,10" --strategy 策略模板 --start 20240101 --end 20241231
```

**说明**:
- 格式: `参数名:值1,值2,值3`
- 多个参数用空格分隔: `threshold:6,8,10 stop_loss_pct:0.02,0.03`
- 系统会生成所有参数组合并逐一回测

**输出示例**:
```
================================================================================
参数扫描结果对比
================================================================================
 threshold   avg_return  sharpe_ratio   win_rate  total_trades
--------------------------------------------------------------------------------
      6.0       12.34%        1.23       62.34%            234
      8.0       15.67%        1.45       65.12%            189
     10.0       11.23%        0.98       58.45%            156
================================================================================
最优参数: threshold=8.0, 平均收益=15.67%
```

---

### 1.3 Walk-Forward 分析 (Walk-Forward Analysis)

**用途**: 验证策略参数的稳健性，避免过拟合

**使用场景**:
- 生产部署前的验证阶段
- 评估策略在未见数据上的表现

**命令**:
```bash
python backtest/strategy_backtest/batch_backtest_V3.py -l 500 --walk-forward "train:6months test:3months" --strategy 策略模板 --start 20230101 --end 20241231
```

**说明**:
- 格式: `train:Xmonths test:Ymonths`
- 训练期6个月: 用于确定最优参数
- 测试期3个月: 用于验证参数的泛化能力
- 窗口滚动前进，重复实验

**输出示例**:
```
================================================================================
Walk-Forward分析结果: In-Sample vs Out-of-Sample
================================================================================
                 训练期                  测试期          训练收益        测试收益        衰减率
--------------------------------------------------------------------------------
      20230101-20230630      20230701-20230930        14.56%        12.34%        -15.25%
      20230701-20231231      20240101-20240331        16.78%        13.45%        -19.84%
      20240101-20240630      20240701-20240930        15.23%        11.89%        -22.01%
================================================================================
      平均                    -        15.52%        12.56%        -18.87%
```

## 2. 标准工作流程

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                           策略开发标准流程                                     │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│   ┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐    ┌────────┐│
│   │ 策略开发  │ -> │ 批量回测  │ -> │ 参数扫描  │ -> │ Walk-    │ -> │ 生产   ││
│   │          │    │ 小样本    │    │          │    │ Forward  │    │ 部署   ││
│   │          │    │ (-l 20)  │    │          │    │ 验证     │    │        ││
│   └──────────┘    └──────────┘    └──────────┘    └──────────┘    └────────┘│
│         │              │              │               │               │       │
│         v              v              v               v               │       │
│   实现策略类      快速验证      寻找最优参数      验证稳健性        达标后   │
│   继承Base       逻辑正确性    确定阈值范围      衰减率<20%       执行交易 │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

**流程说明**:

1. **策略开发**: 继承 `BaseStrategy`，实现 `calculate_score()`, `buy_condition()`, `sell_condition()`
2. **批量回测(小样本)**: 用20-30只股票快速验证策略逻辑
3. **参数扫描**: 扩大样本到100只，系统搜索最优参数
4. **Walk-Forward验证**: 使用更大样本(500+)，验证参数稳健性
5. **生产部署**: 衰减率 < 20% 可投入生产

## 3. 命令参考

### 3.1 完整参数列表

```bash
python backtest/strategy_backtest/batch_backtest_V3.py [参数]

# 股票选择
-l, --limit <数量>        从数据库获取的股票数量
-i, --industry <行业>      行业过滤
-s, --stocks <代码...>     指定股票代码列表
-f, --stock-file <文件>   股票代码文件（每行一个代码）
-S, --stock <代码>        单股票模式（仅用于调试）

# 回测配置
--start <日期>            开始日期 YYYYMMDD (默认: 20250101)
--end <日期>              结束日期 YYYYMMDD (默认: 20251231)
--cash <金额>             初始资金 (默认: 100000)
--strategy <名称>          策略文件名 (默认: 天宫B1策略v1)
--threshold <值>           策略阈值 (默认: 8.0)

# 高级功能
--param-grid <网格>        参数扫描，如: threshold:6,8,10
--walk-forward <配置>      Walk-Forward分析，如: train:6months test:3months
--output-json <路径>      JSON输出文件路径

# 其他
--no-save                 不保存到数据库
-o, --output <目录>        结果输出目录
--use-registry            使用Registry加载策略 (默认启用)
--no-registry             禁用Registry，使用旧方法加载策略
```

### 3.2 常用命令示例

**批量回测**:
```bash
# 测试100只股票
python backtest/strategy_backtest/batch_backtest_V3.py -l 100 --strategy 策略模板

# 指定日期范围
python backtest/strategy_backtest/batch_backtest_V3.py -l 100 --start 20240101 --end 20241231

# 指定行业
python backtest/strategy_backtest/batch_backtest_V3.py -l 100 -i 电子 --strategy 策略模板
```

**参数扫描**:
```bash
# 扫描单个参数
python backtest/strategy_backtest/batch_backtest_V3.py -l 100 --param-grid "threshold:6,8,10" --strategy 策略模板

# 扫描多个参数
python backtest/strategy_backtest/batch_backtest_V3.py -l 100 --param-grid "threshold:6,8,10 stop_loss_pct:0.02,0.03,0.05" --strategy 策略模板
```

**Walk-Forward**:
```bash
# 标准6+3窗口
python backtest/strategy_backtest/batch_backtest_V3.py -l 500 --walk-forward "train:6months test:3months" --strategy 策略模板

# 自定义窗口
python backtest/strategy_backtest/batch_backtest_V3.py -l 500 --walk-forward "train:12months test:1months" --strategy 策略模板
```

**JSON输出**:
```bash

python backtest/strategy_backtest/batch_backtest_V3.py -l 100 --output-json results.json --strategy 策略模板
```

## 4. 最佳实践

### 4.1 样本量选择

| 阶段 | 样本量 | 目的 |
|------|--------|------|
| 开发验证 | 20-30 | 快速迭代，验证逻辑 |
| 参数优化 | 100 | 足够样本找到稳定参数 |
| Walk-Forward | 500+ | 验证稳健性，避免过拟合 |

### 4.2 参数扫描建议

1. **参数范围**: 先大步搜索(如6,8,10)，再小步精调(如7,7.5,8,8.5,9)
2. **避免过度优化**: 参数数量不超过3个，避免组合爆炸
3. **关注泛化能力**: 训练集和测试集差距要小

### 4.3 Walk-Forward评估标准

| 衰减率 | 评估 | 建议 |
|--------|------|------|
| < 10% | 优秀 | 可直接投入生产 |
| 10-20% | 良好 | 可投入生产，需监控 |
| 20-30% | 一般 | 需重新审视策略 |
| > 30% | 差 | 存在过拟合，不建议生产 |

**衰减率计算**: `(测试集收益 - 训练集收益) / 训练集收益`

### 4.4 注意事项

1. **始终使用批量回测验证**: 不要用单股票回测作为策略有效性的依据
2. **单股票模式仅用于调试**: `-S` 或 `-s` 参数仅在看某个具体股票详细日志时使用
3. **保持策略简洁**: 复杂策略往往过拟合风险更高
4. **定期重新验证**: 市场风格变化时需重新进行Walk-Forward分析

### 4.5 禁止事项

- **禁止**用单股票回测结果作为策略上线的依据
- **禁止**在训练集上过拟合参数
- **禁止**移除 `run_backtest()` 后直接改用单股票模式验证策略

## 5. 策略文件规范

策略文件(如 `strategies/策略模板.py`)应只包含:
- `STRATEGY_METADATA` 字典
- `@register` 装饰器
- 策略类(继承 `BaseStrategy`)
- Helper方法

**不应包含**:
- `run_backtest()` 函数
- backtrader/duckdb/pandas 的 import(除非策略确实需要)

这确保策略专注于信号生成，回测逻辑统一由 `batch_backtest_V3.py` 处理。
